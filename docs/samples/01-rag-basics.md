# RAG 基础知识

## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合**信息检索**和**文本生成**的 AI 技术架构。

> [!note] 核心原理
> 先从知识库中检索相关文档片段，再将片段作为上下文注入大语言模型（LLM）的 Prompt 中，让模型基于真实数据生成准确回答。

## 为什么需要 RAG

1. **解决幻觉问题**：LLM 有时会编造不存在的信息，RAG 将回答锚定在真实文档上
2. **知识时效性**：LLM 的训练数据有截止日期，RAG 可以检索最新文档
3. **可溯源**：每个回答都能追溯到具体来源

## RAG 的关键组件

### 1. 文档分块
将长文档切分为可检索的片段。分块策略直接影响检索质量：
- 太小：丢失上下文
- 太大：检索精度下降

推荐以 512 tokens 为起点，根据文档类型调整。

### 2. 向量化
使用 Embedding 模型（如 BGE-M3）将文本转为高维向量。语义相似的文本，其向量在空间中距离更近。

### 3. 向量检索
通过余弦相似度或欧氏距离找到最相关的文档片段。常用工具有：
- **LanceDB**：嵌入式列式数据库，无需独立服务
- **FAISS**：Meta 出品，工业级可靠性
- **ChromaDB**：轻量级，适合原型

### 4. Prompt 组装
将检索结果与用户问题组合，注入 LLM 的上下文窗口。

## 常见应用场景

| 场景 | 描述 |
|------|------|
| 个人知识库 | 搜索自己的笔记、文档 |
| 企业文档问答 | 基于内部文档回答员工问题 |
| 客服系统 | 基于知识库自动回复 |
| 学术研究 | 基于论文库生成文献综述 |
